# ServiceNow ITSM Lab — Project Overview

This project demonstrates hands‑on experience with **ServiceNow IT Service Management (ITSM)** and core **ITIL processes** using a Personal Developer Instance (PDI).  
The lab walks through real‑world workflows including **Incident Management, Service Requests, Change Management, Approvals, Reporting, and Catalog Design**.

This project is designed as a portfolio‑ready example of practical ServiceNow administration skills.

---

## 🎯 Objectives

This lab showcases the ability to:

- Navigate and configure core ServiceNow ITSM modules  
- Create, assign, and resolve incidents following ITIL best practices  
- Build a functional Service Catalog item with variables  
- Configure a Change Request requiring approval  
- Generate meaningful ITSM reports for operational visibility  
- Understand how ITIL concepts map to ServiceNow modules  

---

## 🧩 Lab Components

### **1. Personal Developer Instance Setup**
A free ServiceNow PDI was provisioned through the ServiceNow Developer Portal.  
This instance serves as the environment for all configuration and testing.

---

### **2. Incident Management**
Created and worked a real‑world incident scenario:

- User unable to access Outlook  
- Categorized and prioritized according to ITIL standards  
- Added work notes, troubleshooting steps, and resolution  
- Closed with user confirmation  

**Skills demonstrated:** 
* Incident lifecycle management
* Ticket documentation
* User communication
* Resolution tracking
* ITIL Incident Management process

---

### **3. Service Catalog Item**
Built a fully functional **New Laptop Request** catalog item:

- Custom form with variables (Requester, Justification, Required Date, Model Preference)  
- Assigned to the IT Hardware fulfillment group  
- Designed for new‑hire and replacement workflows  

**Skills demonstrated:**
* Service Catalog administration
* Self-service request design
* Form customization
* Request fulfillment planning

---

### **4. Change Management & Approvals**
Created a Standard Change Request for a Windows security patch deployment:

- Added risk, impact, scheduling, test plan, and backout plan  
- Submitted for approval and processed the approval workflow  
- Moved the change to Scheduled state  

**Skills demonstrated:**
* Change lifecycle management
* Risk assessment
* Change approval workflows
* Deployment planning
* ITIL Change Management process

---

### **5. Reporting & Dashboards**
Generated multiple ITSM reports:

- **Incident Volume by Priority (Last 30 Days)**  
- **Mean Time to Resolution (MTTR)** by Assignment Group  
- **Open Incidents by Assigned Agent**  

**Skills demonstrated:**
* ServiceNow reporting
* Operational analytics
* KPI tracking
* Service performance monitoring

## 📘 ITIL Concepts Demonstrated

| ITIL Concept | Description | ServiceNow Module |
|--------------|-------------|-------------------|
| **Incident** | Restore service quickly after an interruption | Incidents |
| **Problem** | Identify and eliminate root causes | Problems |
| **Change** | Planned modifications with controlled risk | Changes |
| **Service Request** | User request for something new | Service Catalog |
| **SLA** | Response/resolution time commitments | SLA Definitions |
| **CMDB** | Asset and relationship database | Configuration Items |
| **Knowledge** | Articles for known issues | Knowledge Base |
